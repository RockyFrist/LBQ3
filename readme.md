# 冷兵器战斗系统设计文档

> 核心定位：以体力博弈为核心，极简操作，闭环克制
> 设计参考：只狼（架势/弹刀节奏）+ 荣耀战魂（轻重攻击/变招佯攻）+ 异人之下（闪避机动性/普攻克招架）
> 技术栈：Vite 6.x + HTML5 Canvas 2D + Vanilla JS ES Modules
> 视角：纯俯视2D，圆形色块 + 朝向指示线

---

## 一、操作方案

| 操作 | 按键 | 说明 |
|------|------|------|
| 移动 | WASD | 八方向移动 |
| 朝向 | 鼠标 | 角色朝向跟随鼠标 |
| 轻击 | **左键单击** | 出手快，可连按最多3下组成连击串 |
| 重击 | **右键单击** | 蓄力0.70秒，伤害高，附带霸体 |
| 招架 | **空格按住** | 触发格挡判定，松手后有0.55秒后摇 |
| 闪避 | Shift + 方向 | 消耗1体力，朝指定方向翻滚 |
| 变招 | 攻击中按 其他操作键 | 消耗2体力，可在轻击/重击/招架间切换 |

### 变招详细

| 操作 | 效果 | 消耗 |
|------|------|------|
| 轻击中按右键 | 轻击重击 | 2体力 |
| 轻击中按空格 | 轻击招架 | 2体力 |
| 重击中按左键 | 重击轻击 | 2体力 |
| 重击中按空格 | 重击招架 | 2体力 |
| 攻击中按Shift | 闪避取消（不算变招） | 1体力 |
| 命中后 | 不可变招 |  |

### 系统按键

| 按键 | 功能 |
|------|------|
| E | 刷出敌人（对战模式） |
| R | 重置战场 |
| 1-5 | 切换AI难度（新手/普通/熟练/困难/大师） |
| 6 | 拼刀训练模式 |
| 7 | 格挡训练模式 |
| H | 打开/关闭帮助 |
| ESC | 返回主菜单 |

## 二、游戏模式

### 对战模式
- 玩家 vs AI，键鼠操作
- 进入前选择AI难度（1-5）
- 游戏中可按数字键随时切换难度

### 斗蛐蛐
- AI vs AI 观战模式
- 选择双方AI难度，观看对决
- 战斗结束后胜者会远离败者（胜利行走）

### 自动测试
- 批量AI对战数据统计
- **视觉测试**：x8倍速可视化，可观察过程
- **纯数据测试**：瞬间跑完数百轮，只看统计结果
- 选择双方难度和轮数

### 无头测试（Node.js）
```
node test-runner.js --rounds 100 --diffA 5 --diffB 5        # 文本输出
node test-runner.js --rounds 300 --diffA 5 --diffB 5 --json  # JSON输出
node test-runner.js --rounds 50 --detail                     # 每轮明细
```

## 五、招架与格挡

### 招架基础
- 按住空格进入招架姿态
- 招架中暂停体力恢复2.5秒
- 松手后0.55秒后摇（可被攻击抓后摇）
- 招架状态下无法闪避

### 格挡判定（三级精度）

| 等级 | 窗口 | 自身消耗 | 敌方体力扣除 | 反击前摇 | 敌方硬直 | 冻结帧 | 慢动作 |
|------|------|---------|-------------|---------|---------|--------|--------|
| **精准格挡** | 攻击命中前0.12秒 | 0 | 2 | 0.12s | 0.40s | 0.18s | x0.15 持续1.0s |
| **半精准格挡** | 已在招架中 | 0 | 1 | 0.22s | 0.45s | 0.12s | x0.15 持续1.0s |
| **非精准格挡** | 时机偏差大 | 2（自身） | 1（对方） | 0.38s | 0.50s | 0.06s | x0.30 持续0.5s |

### 格挡加速增益
格挡成功后下次攻击前摇压缩：
| 等级 | 速度倍率 | 持续 |
|------|---------|------|
| 精准 | x0.55 | 0.80秒 |
| 半精准 | x0.55 | 0.80秒 |
| 非精准 | x0.75 | 0.60秒 |

### 格挡反击乒乓
- 格挡成功后可按攻击键反击，对方可再格挡，形成连续拆招
- 每轮反击速度增加40%前摇（递减）
- 反击射程65px，弧度63deg，伤害15

## 六、核心克制链

```
重击(霸体) -> 克制 -> 轻击(连击) -> 克制 -> 招架(格挡) -> 克制 -> 重击
闪避 -> 第四维度：重置局面 / 完美闪避
```

### 重击克轻击
- 霸体无视轻击干扰，蓄力中不会被轻击中断
- 重击命中后对方吃高伤害+大击退

### 轻击克招架（渐进破防）
| 连击段 | 打到招架方的效果 |
|--------|----------------|
| 第1下 | 招架方-1体力，不破防 |
| 第2下 | 招架方-1体力，不破防 |
| **第3下横扫** | **破防！** 对方进入0.50秒硬直 |

### 招架克重击
- 格挡重击后扣对方体力（按精度等级）
- 可发起格挡反击进入乒乓

## 七、闪避系统

| 参数 | 数值 |
|------|------|
| 消耗 | 1体力 |
| 持续 | 0.36秒 |
| 速度 | 370 px/s |
| 无敌帧 | 前0.18秒 |
| 完美闪避窗口 | 0.12秒 |

### 完美闪避
- 闪避前半段恰好躲过攻击则触发完美闪避
- 退还体力消耗（等于免费闪避）
- 可自由追击反打

### 完美闪避 vs 精准格挡

| 对比 | 精准格挡 | 完美闪避 |
|------|---------|---------|
| 体力成本 | 0 | 0（退还） |
| 收益 | 敌方-2体力+快速反击 | 免费闪避+自由追击 |
| 后续 | 可能进入乒乓拆招 | 回到中立/追击 |
| 定位 | 正面刚，连续拆招 | 侧面切，拿确定伤害 |

## 八、特殊机制

### 拼刀（轻击 vs 轻击同时命中）
- 双方武器弹回，推开18px
- 硬直0.25秒
- 冻结帧0.15秒
- 慢动作x0.30 持续0.35秒
- 拼刀后立即回到决策点，可形成"叮叮叮"节奏交锋

### 弹刀（重击 vs 重击碰撞）
- 双方各扣1体力
- 弹开90px
- 硬直0.40秒
- 冻结帧0.20秒
- 慢动作x0.20 持续0.50秒

### 处决
- 条件：体力=0时被攻击命中
- 伤害：最大血量的25%（25HP）
- 持续：1.0秒动画
- 处决后恢复3点体力

## 九、AI 系统

### 难度等级（1-5）

| 参数 | 1新手 | 2普通 | 3熟练 | 4困难 | 5大师 |
|------|--------|--------|--------|--------|--------|
| 反应概率 | 35% | 50% | 62% | 82% | 97% |
| 闪避概率 | 8% | 14% | 20% | 30% | 38% |
| 思考CD | 0.35s | 0.22s | 0.16s | 0.06s | 0.02s |
| 攻击率 | 35% | 42% | 50% | 60% | 68% |
| 重击率 | 12% | 20% | 26% | 35% | 42% |
| 最大连击 | 1 | 2 | 3 | 3 | 3 |
| 格挡时长 | 0.25s | 0.38s | 0.50s | 0.65s | 0.85s |
| 重击反应 | 30% | 50% | 65% | 90% | 99% |
| 惩罚率 | 10% | 25% | 35% | 72% | 95% |
| 变招概率 | 3% | 8% | 18% | 45% | 65% |

### 训练模式
| 模式 | 说明 |
|------|------|
| 6拼刀训练 | AI只用轻击对攻，练习拼刀节奏 |
| 7格挡训练 | AI用重击+招架，练习格挡乒乓 |

### AI 特性
- **HTN计划系统**：多步策略链（格挡->反击、闪避->追击、压制连招等）
- **行为预测**：追踪玩家重击/变招率，动态调整应对策略
- **重击对冲博弈**：双方蓄力时AI会根据对手变招率决定是否变招
- **格挡加速利用**：高难度AI会立刻利用格挡加速增益反击
- **拼刀后策略**：拼刀硬直恢复后立即规划下一步（再攻/格挡/闪避）
- **硬直预输入招架**：拼刀被弹中预判对方攻击并缓冲格挡
- **胜利行为**：战斗结束后胜者远离败者，模拟真人感

## 十一、技术架构

```
src/
  main.js              # 入口，菜单->游戏循环
  style.css            # UI样式
  core/
    constants.js       # 所有数值常量
    utils.js           # 纯数学工具
    input.js           # 键鼠输入管理
    camera.js          # 屏幕震动
  combat/
    fighter.js         # 角色状态机（idle/attack/block/dodge/stagger...）
    combat-system.js   # 战斗判定（命中/格挡/拼刀/处决）
    player.js          # 玩家输入->命令映射
  ai/
    enemy.js           # AI行为主逻辑（状态机 + 决策）
    ai-config.js       # AI难度配置（D1-D7参数表）
    ai-plans.js        # HTN计划系统（多步策略链 + 训练行为）
  render/
    renderer.js        # Canvas 2D 渲染
    particles.js       # 粒子系统（火花/血/处决）
  ui/
    ui.js              # HUD（血条/体力/状态/战斗日志）
    menu.js            # 多页菜单（主页/对战/观战/测试/武圣）
  nn/
    nn-agent.js        # 神经网络代理（状态提取 + 动作映射）
    browser-train.js   # 浏览器端NN训练器
  game/
    game.js            # 游戏管理器（模式/胜负/慢动作）
    event-log.js       # 战斗事件→UI/视觉映射
    test-mode.js       # 自动测试模式（统计 + 渲染）
test-runner.js         # Node.js 无头批量测试
train.js               # Node.js 训练脚本
```

### Fighter 状态机
```
idle -> lightAttack -> (combo 1/2/3) -> idle
idle -> heavyAttack -> (startup -> active -> recovery) -> idle
idle -> blocking -> blockRecovery -> idle
idle -> dodging -> idle
blocking -> parryStunned -> idle（被弹恢复）
any -> staggered -> idle（被打硬直）
any -> executing / executed（处决动画）
```
