# 冷兵器战斗系统设计文档

> 核心定位：以体力博弈为核心，极简操作，闭环克制
> 设计参考：只狼（架势/弹刀节奏）+ 荣耀战魂（轻重攻击/变招佯攻）+ 异人之下（闪避机动性/普攻克招架）
> 技术栈：Vite 6.x + HTML5 Canvas 2D + Vanilla JS ES Modules
> 视角：纯俯视2D，圆形色块 + 朝向指示线

## 一、操作方案

| 操作 | 按键 | 说明 |
|------|------|------|
| 移动 | WASD | 八方向移动 |
| 朝向 | 鼠标 | 角色朝向跟随鼠标 |
| 轻击 | **左键单击** | 出手快，可连按最多3下组成连击串 |
| 重击 | **右键单击** | 蓄力0.70秒，伤害高，附带霸体 |
| 招架 | **空格按住** | 触发格挡判定，松手后有0.55秒后摇 |
| 闪避 | Shift + 方向 | 消耗1体力，朝指定方向翻滚 |
| 变招 | 攻击中按 其他操作键 | 消耗2体力，可在轻击/重击/招架间切换 |

### 变招详细

| 操作 | 效果 | 消耗 |
|------|------|------|
| 轻击中按右键 | 轻击→重击 | 2体力 |
| 轻击中按空格 | 轻击→招架 | 2体力 |
| 重击中按左键 | 重击→轻击 | 2体力 |
| 重击中按空格 | 重击→招架 | 2体力 |
| 攻击中按Shift | 闪避取消（不算变招） | 1体力 |
| 命中后 | 不可变招 |  |

### 系统按键

| 按键 | 功能 |
|------|------|
| E | 刷出敌人（对战模式） |
| R | 重置战场 |
| 1-5 | 切换AI难度（新手/普通/熟练/困难/大师） |
| 6 | 拼刀训练模式 |
| 7 | 格挡训练模式 |
| H | 打开/关闭帮助 |
| ESC | 返回主菜单 |

## 二、游戏模式

### 对战模式
- 玩家 vs AI，键鼠操作
- 进入前选择AI难度（1-5）
- 游戏中可按数字键随时切换难度

### 斗蛐蛐
- AI vs AI 观战模式
- 选择双方AI难度，观看对决
- 战斗结束后胜者会远离败者（胜利行走）

### 自动测试
- 批量AI对战数据统计
- **视觉测试**：x8倍速可视化，可观察过程
- **纯数据测试**：瞬间跑完数百轮，只看统计结果
- 选择双方难度和轮数

### 无头测试（Node.js）
```
node test-runner.js --rounds 100 --diffA 5 --diffB 5        # 文本输出
node test-runner.js --rounds 300 --diffA 5 --diffB 5 --json  # JSON输出
node test-runner.js --rounds 50 --detail                     # 每轮明细
```

## 五、招架与格挡

### 招架基础
- 按住空格进入招架姿态
- 招架中暂停体力恢复2.5秒
- 松手后延迟0.12秒 + 0.55秒后摇 = 共0.67秒不可行动
- 招架状态下无法闪避

### 格挡判定（三级精度）

| 等级 | 窗口 | 自身消耗 | 敌方体力扣除 | 冻结帧 | 慢动作 |
|------|------|---------|-------------|--------|--------|
| **精准格挡** | 攻击命中前0.12秒 | 0 | 2 | 0.18s | x0.12 持续1.40s |
| **半精准格挡** | 已在招架中 | 0 | 1 | 0.12s | x0.15 持续1.10s |
| **非精准格挡** | 时机偏差大 | 2（自身） | 1（对方） | 0.06s | x0.30 持续0.50s |

### 格挡后二次博弈

格挡成功后双方进入慢动作，**格挡方短暂停顿后先行解锁，被弹方稍后解锁**，形成有利但不绝对的博弈窗口：

| 参数 | 说明 |
|------|------|
| 攻击方被弹硬直 | 0.10游戏秒（精准格挡下约0.83真实秒） |
| 格挡方行动延迟 | 0.04游戏秒（精准格挡下约0.33真实秒） |
| 格挡加速增益 | 格挡方下次攻击前摇压缩（精准x0.80/半精准x0.85/非精准x0.90） |
| 增益持续 | 精准/半精准0.80秒，非精准0.60秒 |

**博弈选择**：
- 格挡方：利用加速增益出轻击（安全）或重击（高风险高收益）
- 被弹方：格挡（防御）、闪避（重置）、攻击（赌对方不出招）
- 格挡方有约0.5真实秒的先手优势，但不保证命中

### 格挡防误触
- 格挡成功后自动抑制Space输入（blockSuppressed），防止误入blocking
- 松开Space后恢复正常格挡能力

### 格挡拉近
| 等级 | 拉近距离 | 滑动时间 |
|------|---------|---------|
| 精准 | 35px | 0.20s |
| 半精准 | 25px | 0.20s |
| 非精准 | 0px | - |

- 拉近在武器弹开动画结束后执行（延迟拉近）
- 最小间距保护：约52px，防止贴脸

## 六、核心克制链

```
重击(霸体) -> 克制 -> 轻击(连击) -> 克制 -> 招架(格挡) -> 克制 -> 重击
闪避 -> 第四维度：重置局面 / 完美闪避
```

### 重击克轻击
- 霸体无视轻击干扰，蓄力中不会被轻击中断
- 重击命中后对方吃高伤害+大击退

### 轻击克招架（渐进破防）
| 连击段 | 打到招架方的效果 |
|--------|----------------|
| 第1下 | 招架方-1体力，不破防 |
| 第2下 | 招架方-1体力，不破防 |
| **第3下横扫** | **破防！** 对方进入0.50秒硬直 |

### 招架克重击
- 格挡重击后扣对方体力（按精度等级）
- 进入慢动作二次博弈

## 七、闪避系统

| 参数 | 数值 |
|------|------|
| 消耗 | 1体力 |
| 持续 | 0.36秒 |
| 速度 | 370 px/s |
| 无敌帧 | 前0.18秒 |
| 完美闪避窗口 | 0.12秒 |

### 完美闪避
- 闪避前半段恰好躲过攻击则触发完美闪避
- 退还体力消耗（等于免费闪避）
- 可自由追击反打

### 完美闪避 vs 精准格挡

| 对比 | 精准格挡 | 完美闪避 |
|------|---------|---------|
| 体力成本 | 0 | 0（退还） |
| 收益 | 敌方-2体力+加速反击 | 免费闪避+自由追击 |
| 后续 | 慢动作二次博弈 | 回到中立/追击 |
| 定位 | 正面刚，连续拆招 | 侧面切，拿确定伤害 |

## 八、特殊机制

### 拼刀（轻击 vs 轻击同时命中）
- 双方武器弹回，推开18px
- 硬直0.25秒
- 冻结帧0.15秒
- 慢动作x0.30 持续0.35秒
- 拼刀后立即回到决策点，可形成"叮叮叮"节奏交锋

### 弹刀（重击 vs 重击碰撞）
- 双方各扣1体力
- 弹开45px
- 硬直0.40秒
- 冻结帧0.20秒
- 慢动作x0.20 持续0.50秒

### 处决
- 条件：体力=0时被攻击命中
- 伤害：最大血量的25%（25HP）
- 持续：1.0秒动画
- 处决后恢复3点体力

### 预输入缓冲系统
- 在攻击判定、闪避、硬直、被弹、格挡后摇等锁定状态中，输入会被缓冲
- 缓冲窗口：0.35游戏秒
- 回到idle时自动消费缓冲，实现无缝衔接
- 连击链也支持缓冲输入（不会因为按早而断连）

## 九、AI 系统

### 难度等级（1-5）

| 参数 | 1新手 | 2普通 | 3熟练 | 4困难 | 5大师 |
|------|--------|--------|--------|--------|--------|
| 反应概率 | 35% | 50% | 62% | 82% | 97% |
| 闪避概率 | 8% | 14% | 20% | 30% | 38% |
| 思考CD | 0.35s | 0.22s | 0.16s | 0.06s | 0.02s |
| 攻击率 | 35% | 42% | 50% | 60% | 68% |
| 重击率 | 12% | 20% | 26% | 35% | 42% |
| 最大连击 | 1 | 2 | 3 | 3 | 3 |
| 格挡时长 | 0.25s | 0.38s | 0.50s | 0.65s | 0.85s |
| 重击反应 | 30% | 50% | 65% | 90% | 99% |
| 惩罚率 | 10% | 25% | 35% | 72% | 95% |
| 变招概率 | 3% | 8% | 18% | 45% | 65% |

### 训练模式
| 模式 | 说明 |
|------|------|
| 6拼刀训练 | AI只用轻击对攻，练习拼刀节奏 |
| 7格挡训练 | AI用重击+招架，练习格挡二次博弈 |

### AI 特性
- **HTN计划系统**：多步策略链（格挡→反击、闪避→追击、压制连招等）
- **行为预测**：追踪玩家重击/变招率，动态调整应对策略
- **重击对冲博弈**：双方蓄力时AI会根据对手变招率决定是否变招
- **格挡加速利用**：高难度AI会立刻利用格挡加速增益反击
- **二次博弈决策**：格挡被弹后根据对手状态决策（格挡/闪避/攻击）
- **拼刀后策略**：拼刀硬直恢复后立即规划下一步
- **硬直预输入**：拼刀/被击中预缓冲下一步操作
- **攻击承诺系统**：高难度AI有更高概率读招取消攻击
- **胜利行为**：战斗结束后胜者远离败者，模拟真人感

```

  main.js              # 入口，菜单→游戏循环

  core/

  combat/

    player.js          # 玩家输入→命令映射

  ui/
    ui.js              # HUD（血条/体力/状态/战斗日志）
    menu.js            # 多页菜单（主页/对战/观战/测试/武圣）
  nn/

  game/
    game.js            # 游戏管理器（模式/胜负/慢动作/hitFreeze）

```

### Fighter 状态机
```
idle → lightAttack → (combo 1→2→3) → idle
idle → heavyAttack → (startup → active → recovery) → idle
idle → blocking → (linger) → blockRecovery → idle
idle → dodging → idle
blocking → parryStunned → idle (被弹恢复，可预输入)
any → staggered → idle (被打硬直，可预输入)
any → executing / executed (处决动画)
```

### 格挡后时序流程
```
重击命中格挡
  ↓
hitFreeze (双方冻结，玩家输入缓冲)
  ↓
慢动作开始 (x0.12~x0.30)
  ↓
格挡方: parryActionDelay 0.04游戏秒 → 解锁行动 (有parryBoost加速)
攻击方: parryStunned 0.10游戏秒 → idle (可预输入)
  ↓
二次博弈决策点 (格挡方略早解锁，但不保证命中)
```
