# 武器系统设计文档

> 冷兵器对战 (LBQ3) — 多武器扩展规格书  
> 基准版本: v0.8.6 | 武器数量: 5

---

## 一、设计哲学

### 1.1 核心原则

1. **机制差异 > 数值差异**：武器的个性来自独特机制（影步、震慑、距离加成等），而非简单的"伤害高/低"
2. **无无脑策略**：每把武器都有明确弱点，不存在"按一个键就能赢"的打法
3. **反应窗口公平**：所有关键动作（绝技蓄势、重击蓄力）都有足够的视觉预警和反应时间
4. **软克制 ≠ 硬克制**：武器间存在优劣关系（约55:45），但操作可以弥补劣势
5. **刀为基准**：刀保持当前所有参数不变，其他武器围绕它定义"偏移"

### 1.2 参考游戏

| 参考 | 借鉴点 |
|------|--------|
| 哈迪斯 (Hades) | 每把武器完全不同的攻击模式 + 特殊机制，而非只是数值变体 |
| 无间冥庙 (Phantom Blade Zero) | 精确的帧数博弈、弹反系统、武器切换节奏 |
| 只狼 (Sekiro) | 弹反为核心的防御哲学、义手忍具 = 武器特殊机制 |
| 怪物猎人 (Monster Hunter) | 武器间移速/攻速/射程的巨大差异如何通过操作平衡 |

| 速度 | 400 | +30 |

| 完美闪避回复 | +2体力(净) | 双倍 |

#### 特殊机制 — 影步 & 背刺

**影步 (Shadow Step)**
- 触发条件：完美闪避成功时
- 效果：瞬移至对手身后（距离=对手半径+己方半径+5px），朝向自动转为面对对手背部
- 对手获得 `facingLocked` debuff **0.30s**（不能转向，但可以闪避/举盾/攻击）
- 粒子效果：残影 + 烟雾

**为什么需要facingLocked**：AI每帧自动转向最近敌人(`_thinkFacing()`)，如果不锁定朝向，影步瞬间AI就转过来了，背刺窗口为0。0.30s的锁定给予公平但有限的背刺机会。

**背刺加成 (Backstab Bonus)**
- 判定：攻击命中时，命中方向与目标朝向的夹角 < 60°（从正后方来的攻击）
- 效果：伤害 ×1.3
- 适用：轻击、重击、绝技的每一段
- 视觉：金色伤害数字 + "背刺!" 浮字

**核心循环**：完美闪避 → 影步到背后 → 0.30s内从背后连击 → 脱离

**防范无脑策略**：
- 完美闪避窗口仅 0.08s，需要精准timing
- 闪避消耗 1 体力（完美闪避净赚1点，但非完美闪避就是亏1点）
- 影步后对手可以闪避/举盾（只是不能转向），高手可以反向操作
- 射程短(40-48)，必须冒险贴身

#### 绝技 — 影杀

|------|-----|
| startup | 0.35s (可被打断) |

**博弈点**：
- 伤害略高于刀绝技(30 vs 28)，但只能命中单体（刀可以多目标）

- 没有格挡减伤机制（匕首绝技更偏"全中或全miss"的赌博型）

1. **轻击第3段(砸地)有霸体**：即使被轻击打也不中断
2. **破盾强化**：对格挡目标每次命中额外消耗 +1 体力（相当于双倍破盾速度）
3. **命中震动**：所有命中产生更强的屏幕震动（×1.5）

**防范无脑策略**：
- 前2段轻击无霸体，可以被随时打断
- 移速最慢(85%)，容易被放风筝
- 闪避最差，被读到就吃满惩罚
- 重击蓄力 0.90s 有充足的反应窗口

- startup 0.50s + 跳跃0.40s = 对手有 0.90s 逃离时间
- 格挡可以减伤到20（举盾是合理对策）
- 落地后0.50s recovery，若miss则巨大惩罚
- 跳跃阶段有霸体但不是无敌——远程/高射程武器(长枪)反而打不到(跳跃是离地的视觉表现，但判定上不是无敌)

#### 特殊机制 — 后撤刺 & 距离加成

**后撤刺 (Retreat Stab)**
- 触发条件：向后方闪避时（闪避方向与朝向夹角 > 120°）
- 效果：在闪避起始位置自动留下一次向前的刺击判定
- 伤害 6，射程 70px，弧度 35°
- 不消耗额外体力（包含在闪避消耗中）
- 设计意图：阻止对手追击，保持安全距离

**距离加成 (Range Bonus)**
- 目标距离在 65-110px（枪的最佳距离带）命中时：伤害 **×1.15**
- 目标距离在 40px 以内（贴身）时：伤害 **×0.75**
- 适用于：轻击、重击（不适用于绝技和弹反反击）
- 设计意图：奖励保持距离的打法，惩罚被贴身

**防范无脑策略**：
- 弧度极窄(20-50°)，稍微侧移就打不中
- 被贴身后伤害大降(×0.75)，而且射程优势消失
- 后撤刺伤害很低(6)，只是阻止追击不是输出手段

#### 绝技 — 龙舞

| 参数 | 值 |
|------|-----|
| startup | 0.35s (可被打断) |
| active | 0.80s (旋枪) |
| 判定 | 360° AoE，半径 95px |
| 段数 | 4段 (每0.20s一段) |
| 每段伤害 | 6 |
| 总伤害 | 24 |
| 击退 | 每段命中击退 50px |
| recovery | 0.45s |
| 霸体 | active阶段全霸体 |
| 格挡减伤 | 50% (每段3，总12) |

**博弈点**：
- 伤害最低(24)，但核心是**区域拒止**——每段击退50px让对手根本靠不近
- 360° AoE 不需要瞄准，适合被围攻时使用
- 适合在对手靠近时使用——不是为了伤害而是重新创造距离
- 对手的对策：不进入范围等它结束，或者趁startup打断

### 4.5 剑盾 (Sword & Shield) — 防御/反击型

**概述**：坚盾配短剑，全游戏最强防御和弹反能力，但攻击伤害最低。核心循环是精准弹反→反击→盾击控制→磨血。需要极大耐心。

#### 移动

| 参数 | 值 | vs 刀 |

| 基础速度 | ×0.95 (147 px/s) | -5% |

#### 轻击链 (3段)

- 总伤 24 全武器最低——剑盾不靠轻击打伤害
- 射程和节奏中规中矩

#### 重击 — 盾击

|------|-----|-------|
| 蓄力 | 0.55s | 略快 |
| active | 0.16s | - |
| recovery | 0.55s | - |
| 射程 | 45px | 短(近身撞) |
| 弧度 | 80° | 宽 |

- 盾击→轻击combo是剑盾的核心进攻方式

| 参数 | 值 | vs 刀 |
|------|-----|-------|
| 精准弹反窗口 | **0.18s** | +50% |

| 格挡体力消耗 | **0** | 无消耗 |

- **精准弹反窗口0.18s**（vs 刀0.12s）——50%更容易触发精准弹反
- **格挡不消耗体力**——可以一直举盾不怕耗尽
- **精准弹反回弹**：精准弹反时，攻击者自身受其攻击30%伤害（轻击8×30%≈2.4，重击28×30%≈8.4）
- **4次破防**——比标准多1次，更难被打破防御

| 速度 | 370 | 标准 |

| 完美闪避回复 | +1体力(净) | 标准 |

#### 特殊机制 — 盾墙

**盾行 (Shield Walk)**
- 格挡期间可以以 40% 速度移动（其他武器格挡时不能移动）
- 可以举盾缓慢推进，形成持续压力
- 不影响弹反判定（移动中也可以精准弹反）

**防范无脑龟缩**：
- 攻击伤害全武器最低(轻击24/重击12)，纯防守会被时间耗死
- 对手可以拉开距离不交手，逼迫剑盾主动进攻
- 长枪的超远射程可以在格挡范围外安全输出

#### 绝技 — 绝对防御

| 参数 | 值 |
|------|-----|
| startup | 0.20s (举盾姿态预警) |
| 姿态持续 | 1.20s |
| 判定 | 360° 自动格挡（全方位） |
| 首次受击 | **自动精准弹反** |
| 反击伤害 | **35** (强化反击) |
| 反击硬直 | 0.50s |
| 反击射程 | 70px (强化) |
| recovery | 0.30s |
| 无人攻击 | 姿态结束，返还 **60%** 炁 |

**博弈点**：
- **陷阱型绝技**——奖励读心，不是按了就赢
- 对手看到预警(startup 0.20s + 举盾金光特效)后可以选择**不攻击**
- 如果对手不攻击：浪费100炁但返还60—净亏40炁，不算致命
- 如果对手攻击（或已经在攻击中无法取消）：反击35伤 + 0.50s硬直 = 毁灭性反击
- 最佳时机：在对手连击中间释放——对手的中段/末段攻击无法取消，保证触发反击
- 多段攻击（匕首5段连击、刀3段）特别容易被捕获

### 5.1 对局关系

对局优劣以先者视角（55:45为"偏优"，50:50为"均势"）：

```
攻\守     刀       匕首      大锤      长枪      剑盾
刀       50:50    偏劣      均势      偏劣      偏优

剑盾     偏劣     偏优      偏劣      均势      50:50
```

### 5.2 克制逻辑

| 对局 | 优势方 | 原因 |
|------|--------|------|
| 匕首 vs 刀 | 匕首偏优 | 速度碾压中速，影步难以应对 |
| 匕首 vs 大锤 | 匕首偏优 | 速度碾压慢速，大锤追不上匕首 |
| 大锤 vs 长枪 | 大锤偏优 | 霸体无视戳刺，一旦靠近伤害碾压 |
| 大锤 vs 剑盾 | 大锤偏优 | 破盾+1消耗加速磨光体力，盾击挡不住跳砸 |
| 长枪 vs 刀 | 长枪偏优 | 射程压制，刀难以靠近 |
| 长枪 vs 匕首 | 长枪偏优 | 匕首射程太短必须贴身，长枪的后撤刺完美针对 |
| 剑盾 vs 匕首 | 剑盾偏优 | 弹反克多段连击，0.18s窗口抓5段非常容易 |
| 刀 vs 剑盾 | 刀偏优 | 均衡输出vs低输出高防，刀能在剑盾防御间隙打伤害 |
| 长枪 vs 剑盾 | 均势 | 长枪远程vs剑盾防御，互不占便宜 |
| 刀 vs 大锤 | 均势 | 中速vs慢速霸体，看操作 |

### 5.3 绝技交互规则

所有绝技在 **蓄势(startup)** 阶段均可被打断（返还50%炁），与当前规则一致。

| 绝技A \ 状态B | idle | 格挡 | 攻击中 | 绝技active |
|---------------|------|------|--------|-----------|
| 拔刀(刀) | 穿透 | 半伤 | 穿透+锁定 | 绝刀相撞 |
| 影杀(匕首) | 命中 | 穿透(无减伤) | 命中+锁定 | 由对方绝技类型决定 |
| 开山(大锤) | 命中 | 半伤 | 命中 | 由对方绝技类型决定 |
| 龙舞(长枪) | 击退 | 半伤+击退 | 击退 | 击退(打断) |
| 绝对防御(剑盾) | - | - | 反弹35伤 | 反弹35伤 |

#### 异种绝技碰撞（两把不同武器的绝技active同时生效）

| 碰撞情况 | 结果 |
|----------|------|
| 同type绝刀相撞 | 双方弹开(当前已实装，仅刀vs刀) |
| 影杀 vs 任何active绝技 | 影杀被target的霸体/绝技判定打断（匕首绝tech无霸体突入段，命中后才锁定） |
| 开山 vs 龙舞 | 开山落地打断龙舞(更高优先级AoE) |
| 开山 vs 拔刀 | 双方各受伤(拔刀有霸体吃开山伤害，开山有前摇也吃拔刀段伤害) |
| 龙舞 vs 拔刀 | 龙舞击退效果将拔刀推出范围（拔刀可能只命中1-2段） |
| 绝对防御 vs 任何 | 首次命中被自动精准弹反，反击35伤 |

```javascript
// weapon-config.js — 武器配置工厂
{

    special: null,        // 特殊效果 ('aoe360'/'dash'/null)

    blockStaminaCost: 'standard', // 'standard'|'free'|'heavy'
    breakHits: 3,
    counterDamage: 15,
    counterRange: 65,
    counterLunge: 30,
    autoCounter: false,   // 精准弹反是否自动反击
    reflectPct: 0,        // 精准弹反伤害反弹比例
  },

  // === 闪避 ===
  dodge: {
    speed: 370,
    invulnEnd: 0.18,
    perfectStamina: 2,    // 完美闪避体力回复(+1净=消耗1回复2)
  },

  // === 体力 ===
  stamina: {
    max: 5,
    regenInterval: 2.0,
  },

  // === 绝技 ===
  ultimate: {
    type: 'multislash',   // 决定了combat-system的处理逻辑
    // ... 特定参数按type不同
  },

  // === 特殊机制(钩子) ===
  specials: {
    // 每个特殊机制是一个 { type, params } 对象
    // combat-system / fighter 在关键时机检查并执行
  },

  // === VS格挡交互 ===
  vsBlock: {
    staminaCost: 1,       // 每次命中格挡消耗对手体力
    ultimateBlockReduction: 0.50,
  },

  // === AI提示 ===
  aiHints: {
    preferredRange: [45, 65],
    aggressiveness: 0.5,  // 0-1
    dodgeBias: 0.5,       // 0-1
    blockBias: 0.5,       // 0-1
  },

  // === 渲染 ===
  color: '#ffffff',       // 武器主色调
  scale: 1.0,             // 模型大小倍率(不影响判定)
}
```
